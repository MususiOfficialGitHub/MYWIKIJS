Users

Users are the people who are authorized to utilize the Vanguard Financials system services.

All system users have to be first registered as employees of the Institution using the Application, then later captured as users.

NOTE: A user is initially captured into the system as a Customer, then made an Employee and finally a User.

To access this module, click on: System Administration ->Operations -> Security - > Users. 

The following screen opens up:     

The available users are displayed and grouped according to the branch.

The grouping type can always be changed to the preferred one.
 

Creating A User

Click create to open the following screen:

On the above screen:

Employee - Pick the appropriate employee from the Employee Look Up dialog box: 

The employee's Department, Branch, Designation & Email details will be populated to the respective fields.

Username Provider - Select the appropriate username provider from the drop-down list. The username can be entered as user-defined or picked from the Active Directory. 

User Name - in order to type in the username, ensure user-defined is selected.

Role - Check the employee's role from the roles lookup window. 
REMEMBER: A user is only entitled to a single role at any given time. Therefore, several roles cannot be assigned to a single user. 
Once a role is checked, trying to assign another role will automatically relieve the previously selected role. 

Authorized Branches - Select the branch at which the user will be authorized to operate by simply 'putting a tick' on the check box against the branch.  
REMEMBER: A user can only be situated at one branch at any given moment in time. 
It is therefore ideal that only one branch is selected for the user. 

Note: The Usernames should follow a uniform unique format/convention for example the customer's name is Dennis Kiluma Mungai.
The following are examples of the usernames he can use - dennis.mungai or dmungai or dk_mungai or d_mungai. 
The Usernames also have to be in lowercase.

Note: Login credentials as sent to individual users through email and text notifications. Passwords are case sensitive.

 

Once done with the above steps, click on the create button to save the information into the system.

On successful completion you will get a pop-up message confirming that the operation was successful.

 

Editing A User

Identify the user to be edited:

Click on the edit icon against the user to open the following screen:        

The Employee, Department, Branch, Designation, User Name< & E-Mail fields cannot be modified.

Modify the user's role & branch from the Roles lookup & Authorized Branches Tabs respectively. 

 

Click update to commit the changes into the system. 

On successful completion you will get a pop-up message confirming that the operation was successful.

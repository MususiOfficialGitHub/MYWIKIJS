Administrative Divisions

Use this sub module to capture a customer's administrative division into the system.

This is typically used to identify the customer's area of residence.

It is traversed as below : 

 County >> Sub County >> Ward >> Sub Location >> Village. 

To access this sub module click on: Registry Management =>> Navigate =>> Setup =>> Administrative Divisions

The following window will open up : 

The already captured administrative divisions are displayed. 

The traverse tree flag displays all divisions under the main division.

Explanation: When unchecked it filters the list and display only the header entries only, On the contrary when checked it displays the header and their detail records.

NOTE : With regards to the traversal order, all  divisions preceding the village should be captured as headers, and the village as a detail.

This implies that  the village will be the only 'active for selection' entry in this case. 

For Best practice, capture a member's administrative division as his/ her ward according to the current governance structure.

 

Creating An Administrative Division :

Click on the add button to open the following screen :  

On the above screen : 

Creating A Parent Division :

Name - Provide the name of the Parent Division 

Type - Select the type as header entry.

Remarks - Provide appropriate remarks with regards to the entry being captured

 Click create to commit the entry into the system. 

Creating A Detail Division :

Parent- Click on the search icon 

and select the appropriate parent division.

Name - Provide the name of the detail Division 

Type - Select the type as detail entry.

Remarks - Provide appropriate remarks with regards to the entry being captured

 Click create to commit the entry into the system.    

On successful completion you will get a pop up message confirming that the operation was successful.

 

Editing An Administrative Division :

This applies to both header & detail divisions.

Identify  the division to be edited then right click on the entry to see the edit button as shown below:

View Audit log icon -click to view the specific audit log of the entry selected

Click on the edit icon against the division to open the following screen : 

Make appropriate changes to the Parent, name, type & remarks fields. 

Is locked flag-Check is locked to render administrative division locked and not operational

Click update to commit the changes into the system. 

On successful completion you will get a pop up message confirming that the operation was successful.

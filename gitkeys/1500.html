Reports

Use this sub module to view all the system reports.

Click on System Administration -> Navigate -> Operations -> Reports to open the following screen:

All the available reports are displayed. 

Creating A Report 

Click Add New button to open the following window: 

For a Header Entry:  

Parent Report - This field may be left empty. 
Note: A header usually doesn't have a parent identifier and a header should not be its own parent.

Report Name - Type in the report's name into this field. 

Category - Select category as header entry. 

Report Path - This field may be left empty. 
Note: A header report is not viewable so having a report path on a header report doesn't make it viewable.

Report Query - This field may be left empty. 
Note: A header report is not viewable so having a report query on a header report doesn't make it viewable.

For a Detail Entry:

Parent Report - Pick an appropriate parent category from the Report Lookup dialog box:

Report Name - Type in the report's name into this field. 

Category - Select category as detail entry. 

Report Path - Define report's path here. 

Report Query - Define the reports query in this field. 

Create the report into the system.

On successful completion you will get a pop-up message confirming that the operation was successful.

 Editing A Report:

 Identify the report to be edited:

Click on the edit icon against the report to open the following screen: 

On the above screen:

Modify accordingly the report's Parent Report, Report Name, Category, Report Path & Report Query fields.  Click update to commit the update into the system. 

On successful completion you will get a pop-up message confirming that the operation was successful.

Viewing A Specific Report:

 Scroll down to trace a specific report that you want to view or search then retrieve a particular report. 

Once you locate it, click on view.

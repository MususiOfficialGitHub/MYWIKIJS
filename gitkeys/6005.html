G/L Account Determination

Use this sub-module to setup and map the Control accounts to their corresponding G/L accounts.

This process must be done appropriately and carefully for the system to work properly. 

To access this window, click:Accounts Management - > Setup - > G/L accounts - > G/L Account Determination. 

The following window will open up:

How To Map The G/L Accounts

1. As illustrated in the figure below, the system G/L account codes have been predefined. 

 Therefore, using this sub module, you can only map it to another G/L account.

 Select the appropriate System G/L Account Code from the drop down menu:

Using the scroll bar, find the code to be mapped to a G/L Account.

Mapped G/L Account Name - Click on the lookup icon to open the G/L Account Lookup dialog box:

From the G/L Account Lookup dialog box, select the appropriate G/L account that  the code is to be mapped onto

When done click on the UPDATE Button to complete this task. 

On successful completion you will get a pop up message confirming that the operation was successful.

4. Click on the Refresh button to refresh and confirm whether the G/L Account was mapped correctly.

For any determination done in an improper way, there's an option to remove it:

System G/L Account Codes Explained : 

ABC Bank Settlement - This refers to the account that will handle all kinds of settlement actions in reference to services offered by ABC Bank.

  • Agent Commission Settlement - This refers to the account through which settlement of an agent's commission is done,  after transactions have been carried out; applicable in the event of agency banking.
  • Profit and Loss Appropriation - This refers to the account that will handle all the actions related to the appropriation of profit and losses after certain transactions.
  • Deceased Control - This refers to the account through which transactions will be carried out in the event that a member dies.
  • Electronic Funds Transfer Control - This refers to the account that will be handling all transactions carried out involving electronic fund transfers.
  • Employer's Contribution(NSSF) - This refers to the account responsible for handling of the employer's contribution, in particular reference to NSSF.
  • Employer's Contribution(Provident Fund) - This refers to the account that will be used in handling of the employer's contribution, in particular reference to Provident Fund.
  • External Cheques Control - This refers to the account responsible for the handling of external cheques and all actions related to it.
  • External Cheques In Hand - This refers to the account responsible for handling of external cheques in hand and all actions related to them.
  • Fikiwa Settlement(Aggregate) -  Refers to settlements of transactions with regard to wallet top up (agency banking charges/ iprs charges)
  • Fikiwa Settlement(Airtime) -  Refers to settlements of airtime purchase transactions through fikiwa
  • Fikiwa Settlement(USSD) -  Refers to settlement of transactions done through ussd
  • Fikiwa Settlement(SMS) -  Refers to settlement of sms purchase transactions
  • Fixed Deposit -  Refers to the accounts responsible for handling of fixed deposit actions.
  • Fixed Deposit Interest - Refers to the account responsible for the handling of  all interests accrued on fixed deposits.
  • Funeral Rider Expense - Refers to settlement of funeral rider expense payments
  • In-House Cheques Control - Refers to the account which will be used for the handling and control of all in-house cheques.
  • Institution Settlement - Refers to the account which will handle all matters to do with institutional transactions and actions.
  • Interswitch Settlement(POS) - Refers to settlement of Interswitch POS transactions
  • Interswitch Settlement - Refers to settlement of Interswitch ATM transactions
  • Legacy Balances Control - Refers to the account which will handle all ledger balances and related transactions.
  • MCo-op Cash Settlement - Refers to the account which will be handling all actions and transactions related to MCo-op Cash settlements.
  • PesaPepe Settlement (Airtime) - Refers to the account which will handle all transactions related to PesaPepe Settlements, with particular reference to airtime.
  • PesaPepe Settlement (B2C) - Refers to the account which will handle all transactions related to PesaPepe Settlements, with particular reference to Business to Customer.
  • PesaPepe Settlement (C2B) -Refers to the account which will handle all transactions related to PesaPepe Settlements, with particular reference to Customer to Business.
  • PesaPepe Settlement (Centrino) - Refers to the account which will handle all transactions related to PesaPepe Settlements, with particular reference to Centrino.
  • PesaPepe Settlement (SMS) -Refers to the account which will handle all transactions related to PesaPepe Settlements, with particular reference to SMS services.
  • Payables Control - Refers to the account which will handle all transactions with respect to payables control, whether borne by the customer or the institution.
  • Sacco-Link Settlement (POS)- Refers to the account that will be responsible for the settlement of all SaccoLink POS Terminal transactions.
  • Sacco-Link Settlement -  Refers to the account that will be responsible for the settlement of SaccoLink ATM transactions.
  • SpotCash Settlement - Refers to the account that will be responsible for the handling of all transactions in relation to SpotCash.
  • Super Saver Interest -  Refers to the ledger responsible for handling of super saver interest payments.
  • Super Saver Withholding Tax - Refers to the ledger that will hold super saver withholding tax.
  • Third Party Settlement -  Refers to settlement ledger for any other party involved in a transaction
  • Truncated Cheques Settlement -   Refers to the account that will be responsible for handling of cheques, in particular reference to  the Settlement of Truncated Cheques.
  • Wire Transfer Settlement(Non customer) -  Refers to the settlement ledger whenever a wire transfer to non-customers is processed

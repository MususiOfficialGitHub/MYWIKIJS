Operations

Use this sub module to assign a role(s), the rights to carry out the already specified institutional operations within the system in line with his/ her role. 

To access this module, click on System Administration - > Operations - > Access Control Lists - > Operations

The following screen will open up: 

  • To assign a role, click on the System Permission Type to reveal a drop-down menu, see below:

Scroll through the system permission types and identify the permission type to be attached to a role. 

Once you've selected the permission type, the authorized roles will be populated on the Authorized Roles Tab, see below:

Select the role to be assigned the permission type by checking the box against it

For instance, we want to allow the role 'Chief Executive Officer' to have the 'Administrative Customer Viewing' permission. we'll just check against the role:  

Note: You can assign multiple roles to a single Permission Type, and you can also assign multiple permission types to a single role.

Authorized Branches:

You can set some operations to be done at specific branches and by specific roles.

To attach operation permissions to a branch, click on the authorized branches tab: 

The list of available branches will be displayed. 

REMEMBER: we're assigning the roles to branches where they'll be able to carry out the operations, otherwise the permissions will be typically obsolete. 

To assign branches to the roles, check the box against the branch.

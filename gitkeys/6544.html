Discrepancy Matching - Jointly.

         Use this sub-module to jointly match payouts and check off which did not match the accounts numbers. 

this is used to match discrepancies for a single customer who has multiple discrepancies affecting the account over a period of time i.e 

  if  there are discrepancies for account number -001 for the month of January, February and March -  they are corrected using joint discrepancy matching.

To access this window, click : Accounts Module - > Operations - > Recurring Procedures - > Discrepancy Matching -> Matching Jointly. 


 

The following screen will open up :  

Discrepancy Matching -  Jointly Fig 1.


 

   Matching  Multiple Discrepancies :

    On the above screen :

 Full Account Number  - Click on Full Account Number look up icon 

to select a customer's account number from the dialog box : 

Discrepancy Matching -  Jointly Fig 2.

 The associated fields -Customer Type, Membership Status, Membership withdrawal date, Domicile Branch, Employer Name, Duty/work Station, Admin Division, Identification #, Member Number Account Number and  Personal File Number  will be automatically populated with data. 

           The discrepancies tab just below will display all the discrepancies associated with that particular account as shown below: 

 Discrepancy Matching -  Jointly Fig 3.

           Check/ Select the discrepancies to be matched as above. 

Select toggle selection 

 if  all the discrepancies that have been listed are to be matched. 

   The specimen tab displays the customer's particulars for verification purposes. 

       Click on the update button to effect the matching and save the changes into the system. 

On successful completion you will get a pop up message confirming that the operation was successful.

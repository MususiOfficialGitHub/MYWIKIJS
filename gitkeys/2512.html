Cash Requisitions

Use this sub module to make Teller Cash requisition requests.

To access this module, click Front Office Management -> Operations -> Teller -> Cash Requisition.

The following screen opens up:

Tally by count: This option is for keying denominations by count i.e. if a teller/cashier needs notes in 1000's they can key the number of notes they need, the system multiplies the notes by the type of denominations that is selected, example 5x1000,1x200 etc.
Tally by total: This option prompt the teller/cashier to key in the total amount he/she needs i.e. 50,000 this option does not multiply by the denomination select but leaves the amount as entered.

Use the reset button to clear the amount entered. 

Total Amount - Amount to be sent to the teller. This is populated from what is input using Tally-by-Count or Tally-by-Total. 

Primary Description - Enter the Primary Description, for example, name of the teller requesting the cash.

Secondary Description - Enter the Secondary Description, for example, a reason why the teller is requesting the cash.

Reference - Enter the Reference, this could be the date, teller code or something to identify the transaction.

 

Click on the Create button to create the Teller Cash Requisition.

The following message will pop up to notify you that the transaction was successful.

NOTE: The Teller Cash Requisition may now be authorized under Treasury> Cash Requisitions

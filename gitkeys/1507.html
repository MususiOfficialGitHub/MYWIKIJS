Temporary Roles

Use this sub module to assign a role(s) the rights to carry out the already specified institutional operations within the system in line with his/ her role temporarily for a period that is set explicitly. 

To access this module, click on System Administration - > Operations - > Access Control Lists - > Temporary Roles

The default screen shown below will open up on temporary roles sub-module: 

It has a list of Employees who have been assigned temporary roles. It also has an indication of the various particulars in regard to that temporary role assignment.

 

Creating a Temporary Role

To add a new Temporary role assignment, click on the Add New button to open the following tab:

Employee - click on the Employee Lookup as the 1st thing to select the Employee.
The employee's Branch, Designation and Department details will be auto populated.

Start Date - Select the start date at which the temporary role will take effect from.

Start Time - Select the start time at which the temporary role will take effect from.

End Time - Select the end time at which the temporary role will expire.

End Date - Select the end date at which the temporary role will finally expire.

Reason - Enter the reason as to why an employee is being assigned the temporary role.

Temporary Roles:

Select the temporary role the employee will be assigned from the temporary roles tab.

Once done, click on the create button to save the occurrences into the system.

On successful completion you will get a pop-up message confirming that the operation was successful.

Note: You can assign multiple temporary roles to a single employee.


Editing a Temporary Role

On the default list, click on the edit button against the employee record that you wish to edit. 

The edit page should look as shown below: 

The employee details, Branch, Designation and Department will be auto populated.

The Start Date, Start Time, End Time, End Date and Reason will also be populated from what was captured on creating the temporary role.

The Role Name cannot be changed but will be shown from what was captured on creating the temporary role.

Suppressed Permission Details:

Use this tab to restrict system permissions which the temporary role assigned can do.

Select a System Permission Type from the drop down, capture a reason as to why the permission is being suppressed and click Add.

Once done, click on the Update button below to complete the editing process.

On successful completion you will get a pop-up message confirming that the operation was successful.

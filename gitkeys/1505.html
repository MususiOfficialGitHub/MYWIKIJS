Modules

Use this sub module to assign users in a role access rights to specified modules.

A role can have as many tasks as can be assigned in line with institutions need. Be sure to contact relevant authorities & parties before tagging a task to a role.

To access this Module, click on System Administration - > Operations - > Access Control Lists - > Modules.

The following screen opens up: 


All the modules within the system are displayed on the Module drop down:

Select a module to show to submodules; will be shown on below with Setups on the left and Operations on the right.

Identify the module to which access rights are to be granted.

 Locate the submodule whose access rights are to be configured; select it to reveal the roles within the system, under the Authorized Roles pane:

Select the appropriate roles, one by one, that will have access to the sub module.

On clicking on each role, you should get the following pop-up:

REMEMBER: The Setup & Operations are header sub modules. They cannot therefore be selected in this context. 
Multiple roles can be assigned the same sub module's access rights by simply checking all the necessary roles. 
Repeat the above steps for every module to which access rights are to be assigned.

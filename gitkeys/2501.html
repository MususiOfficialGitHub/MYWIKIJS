Cash Transfer

Use this sub-module to perform the following transactions, with particular reference to cash management: 

  • Bank to Treasury
  • Treasury to Teller
  • Treasury to Treasury
  • Treasury to Bank

Treasury's Cash Transfer Requests can also be authorized/rejected from this module.

To access this module, click on Front-Office Management - > Operations - > Treasury -> Cash Transfer

The following screen will be opened:

 

Cash Transfer

There are three operations performed on this module, creating, authorizing and posting a transfer request.

Cash Transfer Tab:
Tally by count: This option is for keying denominations by count i.e. if a teller/cashier has notes in 1000's he can count the notes one by one and key the number of notes he/she has, the system multiplies the notes by the type of denominations that is selected, example 5x1000,1x200 etc.
Tally by total: This option prompt the teller/cashier to key in the total amount he/she has i.e. 50,000 this option does not multiply by the denomination select but leaves the amount as entered.

Use the reset button to clear the amount entered. 

Transaction type: This option is for selecting the type of transaction one is intending to perform i.e cash to teller, bank to treasury - from the drop-down menu options. See below: 

 

Bank To Treasury Transaction Type.

Transaction Type - Select Bank to treasury when receiving money from the bank to treasury.

Bank - Select the target Bank on the search icon.

Treasury - Select the target Treasury on the search icon.
Auto populates with the name of the Treasury of the same branch as the user logged in.

Total Amount - Amount received from the bank to treasury. This is populated from what is input using Tally-by-Count or Tally-by-Total. 

Primary Description - Enter the Primary Description, for example, issuer or something to identify the transaction.

Secondary Description - Enter the Secondary Description, for example, name of the bank the money is issued to or something to identify the transaction.

Reference - Enter the Reference, this could be the cheque number or something to identify the transaction.

 

Click on the Update button to get a prompt to create an authorization request for the transaction.

Click yes if you want to proceed and no if you want to cancel. 

The following message will pop up to notify you that the transaction was successful.

Cash Transfer Requests

Next to the Cash Transfer tab is the Cash Transfer Requests Tab shown below, it's possible to view all the treasury cash transfer requests.

At the top the Cash Transfer Requests Tab, there is a drop down for Sent/Received requests and their Status.

Sent requests are those that originate from the Treasury. For example, Treasury to Bank, Treasury to Teller, Treasury to Treasury (for the originating Treasury).

Received requests are those whose destination is the Treasury. For example, Bank to Treasury, Teller End of Day, Treasury to Treasury (for the destination Treasury).

REMEMBER: Treasury to Treasury requests will for the destination Treasury use ''Received'' while the originating Treasury will be ''Sent''.

NOTE: To Authorize a request, the status must be first be Pending and to post the request, the status must be changed to Authorized. 

 

Cash Transfer Requests (Authorize). 

To authorize a request, use the category (Sent/Received), status (Must be Pending) and date range filters to search for the pending cash transfer request to be authorized. 

After identifying the item, right-click on the item and select the Authorize option.

NOTE: The user who initiated the transaction cannot Authorize it. Trying to do so will result in the following:

Click the Authorize button and the Treasury Management Request Authorization tab will be opened:

Provide the appropriate Remarks and select the appropriate Authorize/Reject action.

Click Update to complete the task.

 

Cash Transfer Requests (Post). 

To post a request, use the category (Sent/Received), status (Must be Authorized) and date range filters to search for the pending cash transfer request to be posted. 

After identifying the item, right-click on the item and select the Post option.

NOTE: The user who authorized a request cannot post the request and must be done by a different person. 

A prompt will be opened after clicking the post option:

Click to Yes/No to accept or decline and the task will be completed.

Click on the Yes button to print the receipt. Click No if you do not wish to print the receipt.

Click on the Yes button to re-print the receipt. Click No if you do not wish to print the receipt.
